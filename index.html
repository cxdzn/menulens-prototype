MenuLens AI Workflow
A real-time accessibility pipeline that sits between confusing restaurant menus and the 1 in 5 people who have dyslexia — powered by Claude claude-sonnet-4.
📷
1 · Menu Capture
Guest scans physical menu via phone camera, or restaurant pushes digital menu through the MenuLens SDK. Text is extracted via OCR or passed directly.
📱 Camera API · Tesseract OCR
🤖
2 · Claude Simplification
Each menu item is sent to Claude claude-sonnet-4 with a prompt tuned for dyslexia-friendly output: short sentences, plain words, no culinary jargon. Allergens, dietary flags, and price parsed simultaneously.
⚡ Anthropic API · claude-sonnet-4-20250514
🖼️
3 · Accessible Rendering
Output rendered using Lexend typeface (purpose-designed for reading ease), wider letter/word spacing, WCAG AA+ contrast, adjustable font scale, and color-coded allergen tags.
🎨 Lexend · WCAG 2.1 AA
🔊
4 · Text-to-Speech
Simplified text is converted to natural-sounding speech via browser TTS or ElevenLabs. Guest can listen to any item without needing to read it at all.
🗣️ Web Speech API · ElevenLabs
📊
5 · CX Analytics Feedback Loop
Every interaction — items viewed, simplifications requested, items skipped, order completion rate — is logged anonymously. The CX dashboard surfaces friction points, satisfaction trends, and accessibility ROI for restaurant operators.
📈 Event stream · Segment · Custom dashboard
